Conference Room ↔ Task Comments · proposed design
Revision 2 · updated from board feedback

One conversation language, two surfaces

The conference room and task/issue comments should feel like the same product. Each surface already owns half of the right design, so the proposal cross-pollinates rather than rebuilds.

Conference room already has the blue/agent bubble language but a stripped-down input. Task comments already have attach + a metadata-row system, but human comments render grey and bot status reads like a comment.

What changed in rev 2 (your feedback):
  • Composer: the essentials are attach and a clear Planning vs Standard mode indicator. Dropped the bold/italic (B) formatting chrome.
  • Bubbles: human → right, bot → left (staggered as you scroll); the blue bubble now sits tight to the human avatar horizontally.
  • Avatars: moved down to hug the tail of each bubble.
Blue is liveness

A human comment, a sent message, an active run — all blue. Blue means "someone or something is live."

Staggered voices

Human bubbles align right by their avatar, agent bubbles align left by theirs — the stagger makes who-said-what scannable.

Status is metadata

"Worked 2m", "updated this task", status changes — quiet rows on a rail, never bubbles.

One simple composer

Same input both places: attach + mode + send. No formatting toolbar. Task adds an assignee; that's the only difference.
